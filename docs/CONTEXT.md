# 灵枢平台领域词汇

本文档定义灵枢平台架构文档中需要稳定使用的领域概念。

## 平台

### 灵枢平台

面向数字化商品生产的供、产、销一体化平台。

平台通过供端提供生产资料，通过产端组织生产流水线，通过销端完成展示、交易和反馈回流。

### 供端

生产资料管理平台。

供端管理 Token、模型额度、基础工具、第三方能力、工具 Adapter、权限、额度、成本和可用性。

供端不直接生产商品，也不直接面向消费者售卖商品。

### 产端

数字化商品工厂。

产端把供端提供的资源和工具组织成可执行、可干预、可质检、可复用的生产流水线。

产端的核心产出是 `ProductPackage`，不是零散文件。

### 销端

商品展示、交易和反馈平台。

销端承接产端输出的 `ProductPackage`，完成展示、购买、下载、评分、评论和反馈回流。

销端不直接修改生产流程。

## 生产

### 数字商品

可以被展示、交付、下载、售卖或授权使用的数字化产物。

第一阶段重点数字商品包括：

- 深度调研报告。
- 视频类商品。

### 产品线

面向某类数字商品的生产配置集合。

产品线包括 Workflow Template、Agent Set、Tool Set、Human Review Points、Quality Gates、Output Schema 和 Product Package Rules。

### Pipeline Harness

产端统一流水线执行框架。

它负责执行节点、保存状态、处理分支、暂停人工审核、恢复任务、触发质检、回退重跑和生成商品版本。

不同产品线复用同一个 Pipeline Harness，通过配置形成差异。

### Workflow Template

产品线的流程模板。

它描述节点顺序、条件分支、回退关系、人工干预点和质量门禁位置。

### 生产任务

一次数字商品生产过程的运行实例。

生产任务从需求输入开始，经过规划、执行、质检、人工干预和商品打包，最终生成一个或多个 `ProductPackage` 版本。

### 节点

Pipeline Harness 中的最小执行单元。

节点可以是智能体节点、工具节点、人工节点、质检节点或打包节点。

### 人工干预

由人对生产任务做出的审核、修改、选择、驳回、重跑或覆盖动作。

人工干预是一等节点，不是流程外补丁。

### 质量门禁

决定节点产出或最终商品是否可以继续向后流转的检查规则。

质量门禁可以输出 `pass`、`manual_review`、`rerun` 或 `fail`。

## 资源与工具

### Token

平台内部资源单位。

第一阶段 Token 主要用于统计成本、控制滥用、设置创作者额度和评估投入产出，不作为用户侧强计费账单。

### 工具能力

产端可请求的抽象能力，例如 `image.generate`、`search.web`、`video.generate`。

产端调用工具能力，不直接依赖具体供应商。

### Tool Adapter

接入具体工具供应商的 Adapter。

Adapter 把供应商差异收敛在供端内部，对产端暴露统一能力。

## 商品与反馈

### ProductPackage

产端输出给销端的标准商品包。

它包含商品元信息、资产引用、质量结果、版权授权、版本信息、价格建议和生产元数据。

### MarketFeedback

销端回传给产端的市场反馈对象。

它包含浏览、下载、购买、评分、评论、退款原因、用户需求和优化建议。

### 商品版本

同一数字商品在生产、优化或反馈驱动重做后形成的可追踪版本。

销端展示的是可上架版本，产端保留生产过程版本和中间资产。
