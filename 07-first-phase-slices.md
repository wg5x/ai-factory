# 第一阶段切片拆解

本文档把第一阶段设计拆成可排期、可验收的工作切片。

当前目标不是开始开发，而是明确第一阶段从设计走向实现时的最小闭环、依赖顺序和验收边界。

## 1. 拆解原则

第一阶段围绕以下闭环拆解：

```text
供端能力可调用
  ↓
产端执行流水线
  ↓
生成 ProductPackage
  ↓
销端展示和交付
  ↓
MarketFeedback 回流
  ↓
触发优化任务
```

切片遵循以下原则：

- 每个切片都应该能独立验收。
- 优先形成端到端路径，避免只做单层能力。
- 先跑通深度调研报告生产线，再评估视频生成生产线的成本和资产策略。
- 第一阶段不实现复杂实时计费、完整分成结算、复杂推荐、插件市场和多团队复杂协作。

## 2. 切片类型

```text
AFK：可以在设计明确后独立推进，不需要过程中频繁确认。
HITL：需要人工确认范围、取舍或演示口径后再推进。
```

## 3. 第一阶段切片

### 3.1 冻结 v0 契约样例

类型：AFK

阻塞依赖：无

目标：

- 为 `ToolCapability`、`ResourceQuota`、`ToolInvocationPolicy`、`ProductPackage`、`MarketFeedback` 提供第一阶段可用样例。
- 让供端、产端、销端可以基于同一组对象理解边界。

验收标准：

- 每个跨端对象至少有一个 JSON 样例。
- 样例覆盖必填字段、可选字段和状态枚举。
- 样例不暴露端内实现细节。

### 3.2 定义第一阶段演示闭环

类型：HITL

阻塞依赖：无

目标：

- 明确第一阶段演示什么、不演示什么。
- 确认深度调研报告主闭环和视频生成简化验证的优先级。

验收标准：

- 明确首个可演示场景。
- 明确视频生成生产线是否作为简化验证项进入第一阶段。
- 明确第一阶段不做复杂销售市场、不做强计费、不做完整分成结算。

### 3.3 供端能力目录与默认调用策略

类型：AFK

阻塞依赖：

- 3.1 冻结 v0 契约样例

目标：

- 产端可以查询可用工具能力、额度和默认调用策略。
- 供端可以返回能力状态、额度状态、默认 provider 和 fallback 策略。

验收标准：

- 至少覆盖 `search.web`、`text.generate`、`report.export`、`content.moderate`。
- 返回 `ToolCapability`、`ResourceQuota`、`ToolInvocationPolicy`。
- 能表达能力禁用、额度耗尽和 fallback provider。

### 3.4 生产任务最小运行骨架

类型：AFK

阻塞依赖：

- 3.1 冻结 v0 契约样例
- 3.2 定义第一阶段演示闭环

目标：

- 创作者可以创建一个生产任务。
- 任务可以从 `created` 流转到 `completed`、`failed` 或 `cancelled`。

验收标准：

- 支持 `ProductionTask` 的最小字段。
- 支持节点执行记录。
- 任务失败时记录失败节点、失败原因和可否重跑。

### 3.5 工具节点调用闭环

类型：AFK

阻塞依赖：

- 3.3 供端能力目录与默认调用策略
- 3.4 生产任务最小运行骨架

目标：

- 产端流水线节点可以调用供端能力。
- 工具调用可以记录额度预留、实际消耗和成本。

验收标准：

- 节点执行前检查额度。
- 工具失败时按 `ToolInvocationPolicy` 重试或 fallback。
- 成本记录绑定到生产任务和节点执行。

### 3.6 质量门禁闭环

类型：AFK

阻塞依赖：

- 3.4 生产任务最小运行骨架

目标：

- 生产任务可以根据质量门禁结果继续、进入人工审核、重跑或失败。

验收标准：

- 支持 `pass`、`manual_review`、`rerun`、`fail`。
- 记录 `checked_items`、`decision`、`reason`、`risk_flags`。
- `rerun` 可以指定 `target_node_for_rerun`。

### 3.7 人工干预暂停与恢复

类型：AFK

阻塞依赖：

- 3.4 生产任务最小运行骨架
- 3.6 质量门禁闭环

目标：

- 人可以在流水线中审核、修改、驳回、选择版本或要求重跑。
- 流水线可以从人工节点恢复。

验收标准：

- 支持 `approve`、`reject`、`edit`、`choose`、`rerun`、`override`。
- 人工修改后的产物产生新的 `output_ref`。
- 人工节点不能绕过最终商品质检。

### 3.8 ProductPackage 生成与版本规则

类型：AFK

阻塞依赖：

- 3.4 生产任务最小运行骨架
- 3.6 质量门禁闭环

目标：

- 产端可以把成品资产、元信息、质检结果和授权信息打成可交付商品包。

验收标准：

- 区分候选包和正式 `ProductPackage`。
- 只有 `completed` 任务关联的正式 `ProductPackage` 可以交付销端。
- 支持 `v1`、补丁版本和基于反馈生成的新主版本。

### 3.9 深度调研报告生产线 MVP

类型：AFK

阻塞依赖：

- 3.3 供端能力目录与默认调用策略
- 3.4 生产任务最小运行骨架
- 3.5 工具节点调用闭环
- 3.6 质量门禁闭环
- 3.8 ProductPackage 生成与版本规则

目标：

- 创作者输入调研主题后，系统可以生成报告类 `ProductPackage`。

验收标准：

- 最小流程覆盖检索、资料归纳、报告写作、事实校验和打包。
- 报告包包含标题、简介、资产、预览资产和质量结果。
- `type_metadata` 包含页数、格式、目录引用和引用摘要。

### 3.10 视频生成生产线 MVP

类型：HITL

阻塞依赖：

- 3.3 供端能力目录与默认调用策略
- 3.4 生产任务最小运行骨架
- 3.5 工具节点调用闭环
- 3.6 质量门禁闭环
- 3.8 ProductPackage 生成与版本规则

目标：

- 创作者输入视频需求后，系统可以按确认模式生成视频类 `ProductPackage`。
- 第一阶段视频线不阻塞报告线主闭环验收。

验收标准：

- 先确认真实视频生成、占位资产或半自动流程的范围。
- 最小流程覆盖脚本、分镜、素材或片段、字幕或配音、打包。
- `type_metadata` 包含时长、分辨率、字幕语言和预览片段引用。

### 3.11 销端接入商品包并展示

类型：AFK

阻塞依赖：

- 3.8 ProductPackage 生成与版本规则
- 3.9 深度调研报告生产线 MVP

目标：

- 消费者可以看到可上架商品列表和商品详情。

验收标准：

- 销端只接收 `ready_for_listing` 商品包。
- 展示信息来自 `type_metadata`、`assets` 和 `preview_assets`。
- 销端维护自己的 `listed`、`delisted` 和 `archived` 上架状态。

### 3.12 预览、下载与基础销售记录

类型：AFK

阻塞依赖：

- 3.11 销端接入商品包并展示

目标：

- 消费者可以预览和下载商品。
- 平台可以记录基础购买和下载数据。

验收标准：

- 购买记录绑定 `product_id`、`package_id` 和 `version`。
- 记录商品收入、作者、版本和分成比例预留字段。
- 已购买用户下载老版本时可以追踪到购买时版本。

### 3.13 评分评论与 MarketFeedback 回流

类型：AFK

阻塞依赖：

- 3.11 销端接入商品包并展示
- 3.12 预览、下载与基础销售记录

目标：

- 消费者反馈可以回流到产端，并绑定具体商品版本。

验收标准：

- 生成 `MarketFeedback`。
- 反馈包含浏览、下载、购买、评分、评论、退款和投诉指标。
- 销端回传反馈时不直接修改 `Workflow Template`。

### 3.14 反馈驱动优化任务

类型：AFK

阻塞依赖：

- 3.8 ProductPackage 生成与版本规则
- 3.13 评分评论与 MarketFeedback 回流

目标：

- 产端可以基于市场反馈经运营确认后创建优化任务，并生成新版本商品包。

验收标准：

- 可以从 `MarketFeedback` 经运营确认创建优化任务。
- 优化任务复用原生产任务上下文。
- 生成补丁版本或新主版本，并可交给销端展示。

### 3.15 端到端追踪与运营查询

类型：AFK

阻塞依赖：

- 3.4 生产任务最小运行骨架
- 3.5 工具节点调用闭环
- 3.8 ProductPackage 生成与版本规则
- 3.13 评分评论与 MarketFeedback 回流

目标：

- 开发和运营人员可以通过后台查询或轻量视图查看一次商品生产从工具调用到销售反馈的完整链路。

验收标准：

- 可以按商品、版本、生产任务、节点、成本、质检和反馈追踪。
- 可以定位高成本节点、失败节点和反馈较差的商品版本。
- 不要求第一阶段提供完整运营仪表盘、复杂经营分析或推荐系统。

## 4. 建议推进顺序

第一批先处理：

1. 3.1 冻结 v0 契约样例。
2. 3.2 定义第一阶段演示闭环。
3. 3.3 供端能力目录与默认调用策略。
4. 3.4 生产任务最小运行骨架。

第二批打通产端主链路：

1. 3.5 工具节点调用闭环。
2. 3.6 质量门禁闭环。
3. 3.7 人工干预暂停与恢复。
4. 3.8 ProductPackage 生成与版本规则。
5. 3.9 深度调研报告生产线 MVP。

第三批接入销端和反馈：

1. 3.11 销端接入商品包并展示。
2. 3.12 预览、下载与基础销售记录。
3. 3.13 评分评论与 MarketFeedback 回流。
4. 3.14 反馈驱动优化任务。
5. 3.15 端到端追踪与运营查询。

视频生成生产线 MVP 应在确认成本、资产策略和演示口径后推进。
